## 1. 게시글 작성 연동 (프론트 ↔ 백엔드)

### 증상
- 게시글 작성 시 "서버 연결에 실패했거나 데이터 전송 오류가 발생했습니다." alert 발생.

### 원인
- 프론트가 호출하는 백엔드 주소(`VITE_API_BASE_URL`)가 설정되지 않음 → `undefined/api/posts` 로 요청됨.
- 프론트엔드가 사용하는 환경변수는 `VITE_API_BASE_URL` 하나뿐 (ChatBot, PostCreate, PostDetail, PostList 4곳에서 사용).

### 조치
- 프론트 루트에 `.env` 생성 후 배포된 백엔드 주소 연결:
  ```
  VITE_API_BASE_URL=https://localhub-backend-dev.onrender.com
  ```
  - ⚠️ 주소 **끝 슬래시(`/`) 제거** — 코드가 `${API_BASE}/api/posts` 로 붙이므로 슬래시 중복 방지.
- `.env` 변경 후 **dev 서버 재시작**(`npm run dev`) 필요.

### 결과
- ✅ 게시글 작성 성공(201) → Render 백엔드 → SQLite 저장까지 정상 연결 확인.

### 메모
- Render 무료 플랜은 15분 미사용 시 슬립. 첫 요청 30~50초 지연 가능(정상).
- 프론트 `.env` 값은 빌드 시 JS 번들에 노출됨 → **OpenAI 키 등 진짜 비밀은 절대 프론트 `.env`에 넣지 않는다**(백엔드 `.env`에만).

## 2. 카카오맵 403 Forbidden 해결

### 증상
- 홈 화면 지도 미표시. `sdk.js` 요청이 **403 Forbidden**.

### 원인 (복합)
1. `index.html`의 카카오 SDK `<script src="...">` URL이 **여러 줄로 쪼개져** 있어 appkey에 개행/공백이 섞여 깨짐.
2. 앱에 **사이트 도메인 미등록**.
3. 앱의 **카카오맵 제품 미활성화(OFF)**.

### 조치
1. `index.html` SDK 스크립트를 **공백/줄바꿈 없이 한 줄**로 수정:
   ```html
   <script src="https://dapi.kakao.com/v2/maps/sdk.js?appkey=<JS_KEY>&libraries=services"></script>
   ```
2. Kakao Developers 콘솔 → 앱(TripPage) → **앱 설정 > 플랫폼 > Web > JavaScript SDK 도메인** 등록:
   ```
   http://localhost:5173
   http://127.0.0.1:5173
   ```
3. Kakao Developers 콘솔 → **제품 설정 > 카카오맵 → 상태 ON(활성화)**.
4. **Ctrl+Shift+R**(캐시 무시 새로고침)로 확인.

### 결과
- ✅ 지도 정상 표시. (`KakaoMap.vue` 컴포넌트 코드는 정상이었고, 원인은 전부 SDK 로딩/콘솔 설정 문제였음.)

### 배포 시 주의
- Netlify 배포하면 그 도메인(`https://XXX.netlify.app`)도 **JavaScript SDK 도메인에 추가 등록** 필요.
